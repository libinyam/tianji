# 安全策略

## 报告漏洞

如果你发现了安全漏洞，请**不要**通过公开 Issue 报告。

请通过以下私有渠道报告：

- 邮箱：contact@tianjihub.cn
- 或使用 GitHub Security Advisory（仓库 Security 标签页 -> Report a vulnerability）

报告时请尽量包含：
- 漏洞的清晰描述和影响范围
- 复现步骤（如适用）
- 你认为的修复建议

## 响应时间

| 阶段 | 时间 |
|------|------|
| 确认收到报告 | 48 小时内 |
| 初步评估 | 7 天内 |
| 修复发布 | 视严重程度，P0 7 天内，P1 30 天内 |
| 公开披露 | 修复发布后 30 天，或与报告者协商 |

## 支持版本

当前仅支持 `main` 分支的最新版本。历史版本不提供安全更新。

## 不在范围内

以下情况请勿通过安全渠道报告，改用普通 Issue：

- 第三方依赖的已知漏洞（请通过 Dependabot alerts 报告）
- 仅在已弃用浏览器中可复现的问题
- 拒绝服务攻击（除非利用了特定代码漏洞）
- 社会工程攻击

## 安全措施

本项目已实施的安全措施：

- 用户写操作通过 CloudBase 数据库安全规则做服务端授权校验
- 管理员操作通过云函数服务端鉴权（`check-admin` / `admin-delete`）
- AI Bot 输入有长度限制和 prompt injection 防御
- 所有用户输入经过 `sanitize.ts` 统一清理
- 前端错误通过 Sentry 上报
- 密钥通过环境变量管理，不硬编码
